## 소프트웨어 테스트 

> 결함 발견 

결함 제거가 아닌 결함의 발견을 목적으로 함 

> 요르돈 법칙

개발 설계시 부터 테스트를 고려, 결함의 조기 발견 및 재유입 방지, 품질 비용 감소 위해  

> 결함 법칙

결함의 80%는 전체 모듈의 20% 내에 발견, 결함이 높은 곳에 자원 집중, 파레토 법칙 때문에 

> 불완전성이란

자원의 한계 때문에 완벽한 테스팅은 불가능, 무한 경로, 무한 입력값, 무한 타이밍 

> 애플리케이션 테스트의 기본 원리 중 상충제 패러독스(Pesticide Paradox)의 의미를 설명하시오? 

동일한 살충제의 반복 사용 시 벌레에게서 내성이 생기는 것에 비유하여, 

동일한 테스트 전략, 기법을 적용할 시 장애가 도출되지 않음을 의미 

> 정황 의존성 

외부요소, 심리요소 때문에 테스트는 테스트 주변환경에 의한 영향을 받음

### 성능 테스트 

> 단위 성능 테스트 : 특정 대상 기능별로 수행하는 테스트 

> 복합 성능 테스트 : 실제 사용자 패턴을 적용하여 태스트 

> 임계 성능 테스트 : 시스템이 최대한 발휘할 수 있는 성능 측정 

> 스파이크 테스트 : 트랜잭션을 동시에 발생시켜 점검

> 확장성 테스트 : 확장 계소를 산정 스케일 아웃 테스트 

> 가용성 테스트 : 특정 부하 조건에서 시스템 안정성 테스트 


### 성능 테스트 구성 

> 대상 시스템 : 소프트웨어, 서버, 인프라, 장비 등 

> 테스트 조직 : 테스트 의뢰자, 테스트 설계자, 테스트 수행원, 비상대응 담당자 등

> 테스트 도구 : 테스트 자동화 도구, 부하 발생 솔루션

> 테스트 스크립트 : 테스트 도구에서 사용되는, 대상 시스템에 맞는 자동화 스크립트

임베디드 시스템 : 제어를 위한 특정 기능을 수행하는 전자 시스템 

IOT : 인터넷이 사물의 모든 네트워크에 연결

디지털 트윈 : 물리적인 사물과 컴퓨터에 동일하게 표현되는 가상 모델

CPS : 물리적인 현상을 관찰하고 계산 및 조작하는 각 물리적 요소들을 제어 하는 복합 시스템 

블록체인 : 블록 이란 분산 데이터를 점대점 방식으로 관리하는 저장 관리 

SSO : 한번의 인증을 통해서 여러개의 서비스를 이용할 수 있는 시스템 

NoSQL : RDBMS를 제외 다른 DBMS를 부르는 용어 

빅데이터 : 페타바이트 크기의 비정형 데이터 

데이터 마이닝 : 빅데이터 안에서 자동적으로 통계적 규칙인 패턴을 찾아내는 기술 

옵티마이저 : SQL을 가장 빠르고 효율적이게 수행할 최적의 경로를 생성해주는 모듈
 
JDBC : 자바에서 데이터 베이스에 접근할 수 있도록 해주는 Access API 

Mybatis : SQL를 사용하여 데이터베이스에 접근할 수 있도록 해주는 Access Framework 

버퍼 오버 플로우 : 메모리에 할당된 버퍼의 양을 초과하는 데이터를 입력하여 악의적인 함수 실행 

비트락커 : window OS 에서 디스크(볼륨) 전체에 암호화를 제공 

인공지능 : 인간의 지능적인 행동과 사고를 컴퓨터 프로그램으로 실현시킨 소프트웨어 

스키마(개념 스키마) : 데이터베이스 구조와 제약조건에 관한 명세 

데이터 사전 : 데이터 항목들에 대한 정보들을 지정한 중앙 저장소 

내부 스키마 : 데이터 베이스의 저장방법을 명세 

외부 스키마 : 사용자 관점에서 본 데이터 베이스 구조 

SAN : 데이터 저장장치와 데이터 서버를 연결해 별도의 LAN을 구성하여 저장 데이터 관리(주로 광 케이블 이용)

인스펙션 : 개발자 외 다른 전문가가 오류를 찾아내는 검토 방법

ARP 스푸핑 :LAN 상에서 자신의 MAC주소를 타인의 MAC주소로 속이는 공격 

코드 난독화 : 역공학 방지를 위해 코드를 알아보기 힘들게 바꿈

ODBC : 데이터 베이스에 접근하기 위한 표준 Access API 

tmpC : 서버의 1분간 최대 처리건수를 나타낸 수치 

회복 테스트 : 소프트웨어 실패를 유도하고 회복이 적절하게 수행되는지 검사하는 테스트 

크라임 웨어 : 온라인 상에서 불법활동을 조장하기 위해 만들어진 컴퓨터 프로그램

BCP : 기업 서비스 연속성 계획 

DRS : 재해 복구 시스템 

APT : 하나의 대상을 목표로 정한 후 내부로 침입을 성공할 때까지 다양한 IT 기술과 공격 

킬 스위치 : 종료가 불가능한 위기 상황에 처한 장치나 기계를 종료하기 위해 사용되는 안전 매커니즘 

다크 데이터 : 저장만 하고 분석에 활용하고 있지 않은 다량의 데이터 

스턱스 넷 : 지멘스의 시스템만을 감염시켜 장비를 제어하고 감시하는 특수한 악성 코드 

PKI : 공개키 암호화 방식으로 전자서명을 제공하는 시스템 

게이트웨이 : 서로 다른 프로토콜끼리 통신이 가능하게 해주는 장치 

REST: 중간 매개체 없이 HTTP 프로토콜로 데이터를 전달하며 리소스를 제공 및 요청하는 프레임 워크 

디피헬만 : 이산대수의 어려움을 이용한ㄴ 키 교환 방식

리치 클라이언트 : SW의 실행을 클라이언트에서 책임

씬 클라이언트 : SW의 실행을 서버에서 책임

URL : 네트워크 상에서 자원이 어디 있는지 알려주기 위한 규약 

3C 분석 : 공객, 경쟁사, 자사 비교 분석

SWOT 분석 : 강점, 약점, 위기, 기회를 토대로 경영 전략 수립

시나리오 플래닝 : 다양한 시나리오 설계 

사용성 테스트 : 사용자가 직접 제품을 사용한 후 보완할 점 파악
 
랜섬웨어 : PC에 있는 중요한 자료를 암호화 한 후, 복호화 프로그램 제공 대가로 피해가에게 돈을 지급하도록 강요하는 악성 코드 

허니팟 : 가상의 정보 시스템을 만들어 놓고 실제로 공격을 당하는 것처럼 보이게 만듬 

DLP : 내부 정보 유출 방지 보안 기술 

조건/결정 커버리지 : 전체 조건식의 결과가 참/거짓을 가지고 개별 조건식도 참/거짓을 갖도록 조합 

결정 커버리지 : 전체 조건식이 참/거짓을 갖도록 조합 

조건 커버리지 : 전체 조건식과 관계없이 개별 조건식이 참/거짓을 갖도록 조합 

시멘틱 웹 : 컴퓨터가 사람을 대신하여 정보를 읽고, 이해하고 가공하여 새로운 정보를 만들어 낼 수 있도록 만든 차세대 지능형 웹 

증강현실 : 실제로 존재하는 환경에 가상의 사물이나 정보를 합성하여 마치 원래의 환경에 존재하는 사물처럼 보이도록 하는 컴퓨터 그래픽 기술 

네트워크 슬라이싱 : 클라우드 환경에서 제공하는 각 서비스에 전용 네트워크 제공 

베타 테스트 : 선발된 잠재 고객으로 하여금 일정기간 무료로 사용하게 한 후에 나타난 여러가지 오류 수정, 보완하는 테스트 

싱클톤 패턴 : 전역 변수를 사용하지 않고 객체를 하나만 생성하도록 하며, 생성된 객체를 어디에서든지 참조할 수 있도록 하는 패턴 

맵 리듀스 : 구글에서 대용량 데이터 처릴르 분산 병렬 컴퓨팅에서 처리하기 위한 목적으로 제작 

브룩스의 법칙 : 이미 늦은 프로젝트에 인력을 투입하면 더 늦어질 뿐임
 
딥러닝 : 여러 비선형 변형 기법을 통해 높은 수준의 추상화를 시도하는 기계학습 알고리즘의 집합
